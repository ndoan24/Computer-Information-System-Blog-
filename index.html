Protocol Info
What is it? Purpose Terms/Vocabulary Examples Visuals

The Internet Protocol (IP) defines unique addresses for a network and a host machine.
The Transmission Control Protocol (TCP) provides IP packets with basic service for safe delivery:

  • error detection
  • safe detection transmission
  • assurance that data are received in the correct order

The User Datagram Protocol (UDP) is the datagram protcol also built on top of IP. Has the same packet-size limit (64KB) as IP. Provides 65, 536 different ports. Connectionless protocol without error. Useful for sending small size data in a repetitive way such as time information.

The purpose of an IP address is to handle the connection between a device and a destination site. The IP address uniquely identifies every device on the internet
  • FTP (File Transfer Protocol) allows the transfer of collection of files between two machines connected to the Internet.
  • Telnet (Terminal Protocol) allows a user to connect to a remote host in terminal mode.
  • NNTP (Network News Transfer Protocol)allows the constitution of communication groups (newsgroups) organized around specific topics
  • SMTP (Simple Mail Transfer Protocol)defines a basic service for electronic mails
  • SNMP (Simple Network Management Protocol) allows the management of the network. FTP Telnet SMTP

IP address is followed by a slash and the number (indecimal) of bits. IPv4 address may be: 192.0 2.1

      Robot, Oct. 27, 2020

      Meal Delivery Cart Robot

      The input for the meal delivery cart is the speaker where it can talk and respond to customers. It can also play music as well when food gets delivered it celebrates with music. The meal delivery cart is also voice activated like Alexa but 2000x better because this is the future. The output is the recording of when the meal delivery carts gets to and from the location of resturant to customer's location. Another output is the GPS location where customers and resturants can keep track of the food. The Central Processing Unit (CPU) is the data and memory work together within the RAM inside the meal delivery cart. The little square processor inside the robot is the "brain". It requires memory to buffer GPS tracking, text to speech, and music playback. The arithmetic and logic unit (ALU) is the calculations the robot does in the GPS to keep track of how long the delivery will take. The control unit is the processor inside the meal delivery cart. This processor tells the robot how to respond to text-to-speech, how fast to move, and to censor when it is safe to cross the street with stop light movement. This is also controlled by the person watching the robot behind the scenes.
